Voraussetzungen 🌈

  • AD DS + DNS laufen auf dem Domänencontroller.
  • Freigaben vorhanden: \\\fileserver\SharedData (S:), \\\fileserver\UserHomes (U:).
  • DNS der Clients zeigt auf den DC.
Hinweis: Ohne korrekten DNS schlägt der Domänenbeitritt fehl.

1 · Netzwerkkonfiguration 💫

GUI

  1. Einstellungen → Netzwerk & Internet → Adapter → IP‑Zuweisung bearbeiten.
  2. IPv4 manuell: IP, Maske, Gateway und DNS = DC.

Check

ipconfig /all
ping dc01.meinedomain.local
nslookup dc01.meinedomain.local

2 · Domänenbeitritt 🐣

GUI

  1. sysdm.cpl → Computername → Ändern → Domäne.
  2. Domänen‑Admin eingeben → Neustart.

PowerShell

Add-Computer -DomainName "MEINEDOMAIN.local" -Credential (Get-Credential) -OUPath "OU=Clients,DC=MEINEDOMAIN,DC=local"
Restart-Computer

3 · Laufwerke S: & U: 🍬

Manuell

  1. Explorer → Dieser PC → Netzlaufwerk verbinden.
  2. S: \\\fileserver\SharedData
  3. U: \\\fileserver\UserHomes\%USERNAME%

Per GPO

  1. GPO „Drive‑Mappings“ → Benutzerkonfiguration → Einstellungen → Windows‑Einstellungen → Laufwerkszuordnungen.

4 · GPOs anwenden 🎀

Aktualisieren

gpupdate /force

Bericht

gpresult /h C:\Temp\gp.html & start C:\Temp\gp.html

5 · Drucker 🖨️✨

  1. GPO → Benutzer/Computerkonfiguration → Einstellungen → Drucker.
  2. Freigabepfad: \\\printserver\OfficePrinter.
